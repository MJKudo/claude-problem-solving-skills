# 要件定義書: (案件名)

## 【必読】作業着手前の指示

本書を受け取ったエージェントは、即座に作業に着手してはならない。まず本書の「問題構造」から「作業手順」までの全節を客観的にレビューし、論理の飛躍・根拠のない前提・曖昧な記述・実現可能性の疑義を抽出せよ。抽出した問題点をユーザーとすり合わせ、解消の合意を得た後にのみ作業に着手すること。

## 【必読】操作上の絶対規則

- 実行・提示するすべての操作手順は、状態の復元手段の事前確保を先頭に含めるか、それが不可能な場合は実施内容・条件・結果を再現できる記録手順とすること
- 稼働中の本番環境への直接の不可逆な操作(削除・上書き・設定変更・再起動・停止・デプロイ等)はいかなる場合も禁止。環境が本番かどうか不明な場合は本番とみなすこと。本番への操作は復元手段を先頭に置いた手順書の提示+ユーザー実施とすること
